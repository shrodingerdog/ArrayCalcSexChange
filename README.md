Ниже код для первого теста выхода в свет плагина для экспорта Listening Planes напрямую из VW. 
Версии, на которых проходили успешные тесты: VW 2026 Update 3 Win, VW 2025 Update 8 Mac. 

Ввиду моей лени, слабого чувства юмора и полного непонимания происходящего внутри кода, есть ряд особенностей по установке и использованию, которые я постарался изложить ниже в единой куче:

1) Установка осуществляется путем вставки кода по следующему пути Tools -> Pluguins -> Pluguin Manager, вкладка Custom Pluguins -> New (мне понравилось имя ArrayCalc SEXChange) ->Command, далее Edit Script, Language -> Pytho nScript. Вставляем код и жмякаем оки. Далее понадобится вытащить кнопку вызова скрипта в удобное место. Предлагаю: Tools -> Workspace -> Edit current Workspace. Слева в Commands можно вбить твое имя плагина (в нашем случае будет достаточно вбить SEX, и он сразу найдет его). Справа раскрываем список, например Tools и переносим плагин в удобное место в списке. Как ты теперь догадался, после сохранения быстрый вызов срипта доступен прямо в выпадающем списке Tools. 

2) Особенности экспортируемых объектов. Данная версия рассчитана на работу с 3D Polygon объектами ТОЛЬКО. Взял я это из своего опыта работы с вектором в подготовительном процессе для ArrayCalc. Обычно я брал 3d poly и обрисовывал нужные мне плоскости в ихначальной модели на новом слое/классе. Далее выгонял эти объекты в скетчап, ну и дальше сам знаешь. 
Теперь достаточно проследить, чтобы все объекты либо изначально были 3d Polygon (окно Object Info справа поможет тебе определить тип объекита), либо требуется их конвертация. Этот процесс доступен по вызову команды Convert to 3d Poly(после нажатия кнопки F, то есть респект - после "ungroup"). Далее достаточно выбрать все объекты и обнаружить в окне Object Info N 3D Ploygons, что будет означать готовность к конвертации. 

3) После выбора всех плоскостей жмякаем на наш скрипт и выбираем Listening Height. В процессе конвертации участвует несколько счетчиков для небольшого дебага. Если значение skipped будет не равно 0, это значит, что нужные плоскости после триангуляции могли быть проигнорированы. Будь внимателен при импорте в AC, плоскостей может не хватать. Я пока с этим не сталкивался, но что делать в случае бага - хз. 

4) Экспорт происходит в автоматически создаваемую папку вслед за диском C. Необходимо это, чтобы обойти вопрос, связанный с кириллицей и возможно русской папкой пользователя. Создаваемая папка называется AC Exp. Ищи ее сразу в корне диска С. Файл имеет имя с системным временем для удобства стека всех файлов и малой возможности их дупликатов.

5) Процесс и алгоритм триангуляции в данной версии простой и незамысловатый, поэтому работает только с выпуклыми многоуголниками (пожалуйста, чекните, что значит выпуклый, а что значит вогнутый многоугольник), постараюсь в след версиях пофиксить. Внутри экспорта, в отличии от AC Exchange, треугольники имеют более аккуратную иерарохию и изначальный цветовой вид, что позволяет пробегаться по ним "стрелочкой" для последующего выбора, группирования и превращения, например, в объект "Обстакл".
